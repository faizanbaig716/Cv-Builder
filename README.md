# CV Builder – Free Resume Generator

A fully client-side CV builder with live preview and one-click PDF download. No backend, no login, no data sent anywhere.

## Files

```
cv-builder/
├── index.html   ← Main page
├── style.css    ← All styles
├── app.js       ← All logic
└── README.md    ← This file
```

## Features

- Personal info, education, work experience, skills, languages, certifications, projects
- Add / delete entries dynamically
- Live preview updates as you type
- One-click PDF download (A4, print-ready)
- Fully responsive (mobile friendly)

## Deploy to GitHub Pages (FREE) — Step by Step

### Step 1 — Create a GitHub account
Go to https://github.com and sign up (free).

### Step 2 — Create a new repository
1. Click the **+** icon → **New repository**
2. Name it: `cv-builder` (or any name you like)
3. Set to **Public**
4. Click **Create repository**

### Step 3 — Upload files
1. Click **Add file** → **Upload files**
2. Drag and drop all 3 files: `index.html`, `style.css`, `app.js`
3. Click **Commit changes**

### Step 4 — Enable GitHub Pages
1. Go to **Settings** tab of your repository
2. Scroll to **Pages** in the left sidebar
3. Under **Source**, select **Deploy from a branch**
4. Choose branch: **main**, folder: **/ (root)**
5. Click **Save**

### Step 5 — Get your live URL
After 1–2 minutes your site is live at:
```
https://YOUR-USERNAME.github.io/cv-builder/
```

## Alternative: Netlify (Even Easier)

1. Go to https://netlify.com → Sign up free
2. Drag the entire `cv-builder` folder onto the Netlify dashboard
3. Your site is live instantly with a URL like `https://random-name.netlify.app`
4. You can rename it to something like `ahmed-cv.netlify.app`

## How to Use

| Tab | What to fill |
|-----|-------------|
| Personal | Name, title, email, phone, location, LinkedIn, summary |
| Education | Degree, institution, year, GPA — add as many as needed |
| Experience | Job title, company, period, bullet-point achievements |
| Skills | Type a skill → press Enter or click Add |
| Extras | Certifications and projects |

**Tip:** In experience descriptions, start lines with `•` or `-` to get bullet points in the PDF.
